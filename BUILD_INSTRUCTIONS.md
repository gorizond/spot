# Инструкции по сборке LCD C++ приложения с GPIO поддержкой

## Обзор

Внесенные изменения добавляют полноценную реализацию GPIO функций для управления LCD дисплеем через GPIO пины Raspberry Pi. Эти изменения корректно работают только на Linux системах с GPIO интерфейсом (в частности, на Raspberry Pi).

## Особенности сборки

### На Raspberry Pi/Linux (целевая платформа)

Приложение будет собрано с полноценной GPIO поддержкой:

```bash
cd /path/to/spot
cd spot_lcd_cpp/build
cmake .. -DENABLE_GPIO=ON -DENABLE_ROS2=OFF
make
```

### На macOS/Windows (платформы разработки)

Приложение будет собрано без GPIO поддержки (в "симуляционном" режиме):

```bash
cd /path/to/spot
cd spot_lcd_cpp/build
cmake .. -DENABLE_GPIO=OFF -DENABLE_ROS2=OFF
make
```

## Требования для работы на Raspberry Pi

1. Установленная библиотека WiringPi:
```bash
sudo apt-get install wiringpi
```

2. Контейнер должен иметь доступ к GPIO пинам (обычно через `/dev/gpiomem`)

3. Права доступа к GPIO (контейнер должен запускаться с соответствующими привилегиями)

## Проверка работоспособности

После деплоя на Raspberry Pi:

1. Убедитесь, что контейнер запускается с привилегиями для доступа к GPIO
2. Проверьте, что пины 25, 24, 23, 17, 18, 22 правильно подключены к LCD дисплею
3. Мониторьте логи контейнера на предмет успешной инициализации WiringPi
4. Проверьте отображение текста на физическом LCD дисплее

## Docker образы

Обновленные Dockerfile обеспечивают правильную установку WiringPi библиотеки и включение GPIO поддержки при сборке на целевых платформах.
